**Root cause**: VFP9 sees `SEEK(...)` at the start of a statement line as the SEEK **command**, making `(expr, alias, tag)` an invalid expression (the comma is unexpected). The fix `= SEEK(...)` forces the parser to treat it as a function call. The `IF !SEEK(...)` occurrences were already correct because expression context forces function interpretation. The `SET RELATION TO IN alias` form is simply not valid VFP9 syntax; only `SET RELATION TO` (clears relation in current work area) works.
